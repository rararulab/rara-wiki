# MkDocs Terminal Theme Request

## Context

After seeing the first custom GitHub Pages implementation, the user rejected the visual
result and requested a switch to "Site built with MkDocs and Terminal for MkDocs."

## Durable Decisions

- The public reading surface should use MkDocs as the static site generator.
- The theme should be Terminal for MkDocs rather than a custom hand-rolled frontend.
- Search should come from the MkDocs built-in search plugin as surfaced by the Terminal
  theme.
- Repository wiki pages remain the source of truth; the public docs tree should be generated.
